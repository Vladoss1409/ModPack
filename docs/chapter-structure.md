# Структура глав CoopTech — шаблон (v0.2)

Зафиксировано по запросу игрока. Продолжение реализации — в следующих сессиях.

## Принцип «волны + ветки»

Каждая **основная глава** (Prologue, Ch1–Ch12) строится **снизу вверх** на вкладке FTB Quests:

```
                    [ Joint finale — gear, крупный ]
                              ↑
                    [ Prep / сюжетный мост ]
                              ↑
                    [ Gate — diamond, 2 из 4 ]
                     ↗  ↗  ↗  ↗
            [ Pool A ] [ B ] [ C ] [ D ]   ← 4 параллельных ветки
                              ↑
                    [ Intro — gear, 1 на главу ]
```

### Обязательные правила

| Элемент | Количество | Механика |
|---------|------------|----------|
| **Старт главы** | 1 | Checkmark или простой крафт; открывает первую волну |
| **Pool-волна** | 4 квеста | Параллельны; разные моды/роли |
| **Gate** | 1 | `dependencies: [4 pool IDs]`, `min_required_dependencies: 2` |
| **Мост / prep** | 1–2 | Сюжет, еда, ресурсы перед финалом |
| **Joint finale** | 1 | Совместный рейд / босс / якорный фрагмент |

После gate открывается **следующая волна** (ещё 4 pool + gate) — квесты «выплывают» из завершённых, линии зависимостей идут вверх.

### Целевый объём на главу

- **Минимум:** 1 intro + 2 волны (8 pool + 2 gate) + prep + finale ≈ **14–16 квестов**
- **Цель для ~140 ч:** 3–4 волны pool + сайд-ответвления ≈ **20–28 квестов** на главу
- **Prologue + Ch1–Ch3** сейчас: ~6–9 квестов — **нужно расширить** до шаблона выше

## Координаты (FTB)

- **Intro:** `x: 0`, `y: 0` (низ вкладки)
- **Pool (4 шт.):** `y: -2`, `x: -4, -2, 2, 4` (или симметрично)
- **Gate:** `y: -4`, `x: 0`, `shape: "diamond"`
- **Следующая волна:** `y: -6`, `-8`, … (вверх = меньше y **или** больше y — держать единый стиль в главе; сейчас: y растёт вниз, финал внизу экрана — **перевернуть на «старт снизу»** при расширении)
